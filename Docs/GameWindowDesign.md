# 小窗口挂机游戏窗口设计

## 目标
- Windows Standalone 默认使用 `320 x 320` 小正方形窗口。
- 大部分时间窗口背景透明，并允许鼠标点击穿透。
- 只有底部状态条和展开后的详情面板可见、可交互。

## 收起态
- 底部保留一条状态栏，高度约 `72px`。
- 状态栏展示当前挂机进程、进度条和一个 `详情` 按钮。
- 状态栏上方展示从左到右排列的 4 名我方单位和 4 名敌方单位，用于表达站位、攻击范围和挂机战斗进行中。
- 鼠标不在任何可见 UI 区域时，窗口保持穿透，玩家可以直接操作下方桌面或其他窗口。

## 展开态
- 点击 `详情` 后，在同一个 `320 x 320` 窗口内展开详情面板。
- 详情面板展示本轮进度、预计结算、当前状态和收益速度等基础信息。
- 按钮文本变为 `收起`，再次点击后隐藏详情面板。

## 交互区域
- 可交互区域包括底部状态栏、战斗示意区、展开中的详情面板。
- 鼠标进入任一可交互区域时，Windows 窗口临时关闭穿透。
- 鼠标离开这些区域后，窗口恢复穿透。
- 所有可见 UI 的非点击控件区域都可以拖拽移动窗口；带点击事件的 UI，例如 `详情` 按钮，只响应点击，不触发拖拽。

## 视觉原则
- 非交互区域不绘制背景，让桌面完全透出。
- 状态栏和详情面板使用半透明深色底，保证信息可读但不打扰桌面。
- UI 先服务功能，不做复杂动效或额外装饰。
